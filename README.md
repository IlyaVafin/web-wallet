# О функционале сайта 
### Авторизация:
  При первом заходе пароль "0000". Состояние авторизован ли пользователь храниться в cookie. При смене пароля пользователя перекидывает на `/login/login.html` и удаляется состояние cookie. Сам пароль хранится в localStorage под ключом password. Есть валидация на неверный пароль.
  Если пользователь не логинился, он не сможет зайти на страницы `/home/home.html` и `change-password/change-password.html`

  
  <img width="831" height="524" alt="image" src="https://github.com/user-attachments/assets/b052eb6f-ec34-404d-8621-bb37529d6b22" />

  
### Формы:
  Есть валидация полей. Использованы стандартные браузерные select и date-picker. При отправке формы в таблице создается новая соответствующая запись

  
  <img width="1593" height="577" alt="image" src="https://github.com/user-attachments/assets/0cad31db-b99d-416e-bce3-0311ac096df9" />

### Фильтрация:
  Фильтрация идет по месяцам, дням, номерам недели месяца (т.е. 1 неделя, 2 неделя и т.д.) и по категориям. Для доходов все те же фильтры кроме категории.

### Таблицы:
  Таблицы содержит всю заполненную информацию из формы. Над таблицами подсчитывается общая сумма расходов и доходов за заданный период времени. Вся информация сохраняется после перезагрузки в localStorage под ключами incomes (доходы) и costs (расходы)

  <img width="1610" height="481" alt="image" src="https://github.com/user-attachments/assets/b2a97645-9d34-4cfb-9ad3-f3a87bc05eb3" />

### Боковая панель:
  Содержит в себе текущий баланс который сохраняется в localStorage и потом берется из него. Кнопка _Выйти из аккаунта_ которая удаляет cookie флаг `isAuthorized` и перенаправляет пользователя на `/login/login.html`. Ссылка _Сменить пароль_ перекидывает на страницу 
  `change-password/change-password.html` где можно сменить пароль.

  <img width="275" height="260" alt="image" src="https://github.com/user-attachments/assets/7476870b-e2dc-45b2-af8b-21b07769c401" />


### Смена пароля:
  Есть простая валидация на не пустую строку (хотя можно было и запариться, но я не стал). После отправки формы новый пароль перезаписывает старый в localStorage и перенаправляет пользователя на `/login/login.html` чтобы он ввел новый пароль. 
  Удаляется флаг `isAuthorized` из cookie.

  - Вычисляем процент от общих трат
